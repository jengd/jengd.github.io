git checkout -- file 撤销文件操作
【 精彩片段 1 】
场景1：当你改乱了工作区某个文件的内容，想直接丢弃工作区的修改时
用命令git checkout -- file。# git checkout -- readme.md

场景2：当你不但改乱了工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步
第一步用命令git reset HEAD <file>，就回到了场景1
第二步按场景1操作。#git reset HEAD readme.md.  git checkout -- readme.md  
（好像直接用  git reset  --hard  xxxxhash值 恢复到指定版本也是可以的）

场景3：已经提交了不合适的修改到版本库时，想要撤销本次提交
参考版本回退一节，不过前提是没有推送到远程库。# git reset  --hard  xxxxhash值

【删除文件 】譬如你在工作区删除了 test.txt，这时候需要执行 git rm text.txt
然后再执行 git commit -m "delete test.txt" 操作
如果是工作区误删除了test.txt文件操作，
这时候可以从版本库里面恢复 ( git  checkout -- test.txt )

【 注意：从来没有被添加到版本库就被删除的文件，是无法恢复的！ 】

与远程 git 仓库结合

1、配置 ssh key 
（#执行ssh-keygen，生成公钥私钥
进入~/.ssh/id_rsa.pub 
把公钥内容复制到 git 服务器的setting ssh_key 位置配置好）

2、添加远程仓库     git remote add origin git@你的远程git仓库地址

3、git push -u origin master 推送到远程服务器
（origin是告诉git推送到远程的意思，之后可以把 -u 参数去掉，git push origin master）

  删除远程库
  如果添加的时候地址写错了，或者就是想删除远程库
  可以用git remote rm <name>命令。使用前，建议先用git remote -v查看远程库信息：

$ git remote -vorigin  
git@github.com:michaelliao/learn-git.git (fetch)origin  
git@github.com:michaelliao/learn-git.git (push)
然后，根据名字删除，比如删除origin：

$ git remote rm origin
此处的“删除”其实是解除了本地和远程的绑定关系，并不是物理上删除了远程库。
远程库本身并没有任何改动。要真正删除远程库，需要登录到GitHub，在后台页面找到删除按钮再删除。

创建合并分支

查看分支：git branch

创建分支：git branch <name>

切换分支：git checkout <name>或者git switch <name>

创建+切换分支：git checkout -b <name>或者git switch -c <name>

合并某分支到当前分支：git merge <name>

删除分支：git branch -d <name>

强制删除分支   git branch -D <name>  （用大 D 哦）
删除远程分支   git push origin --delete <name>
拉取指定分支   git pull <远程库名> <远程分支名>:<本地分支名>  

把新建的本地分支push到远程服务器，远程分支与本地分支同名（当然可以随意起名）：
git push origin develop:develop

冲突解决

当你在 dev 分支与 master 分支同时修改 README.md 时，两者同时执行了 add 跟 commit 操作。
这时切回 master 分支，执行 git merge dev，想把 dev 分支合并到master，就会出现冲突提示。
这时候需要你手动修改圈内的内容以解决冲突，然后在执行 

git add README.md , git commit -m 'xxxxx'  再次提交

bug 分支修复

当手头工作没有完成时，先把工作现场git stash一下.
然后去修复bug，修复后，再git stash pop，回到工作现场；

git cherry-pick 4c805e2    #能复制一个特定的提交到当前分支

在 dev 分支中，如果执行 git push 失败
原因是没有指定本地dev分支与远程origin/dev分支的链接，根据提示，设置dev和origin/dev的链接

则需要执行 git branch --set-upstream-to=origin/dev dev
在执行 git pull

git 打 tag

git tag v1.0

也可以指定 commit id，git tag v0.9  0e8a6002

查看tag 列表，git tag

查看指定tag，git show v1.0

删除tag ，git tag -d v0.1

推送tag到远程，git push origin v1.0

推送全部tag到远程，git push origin --tags

删除远程tag，1、先删除本地tag，git tag -d v0.1    
2、从远程删除，git push origin :refs/tags/v0.1

关联两个仓库

1、先删除 git remote rm origin

2、添加 github 远程仓库，git remote add github git@xxxxx/learngit.git

3、添加 gitee 远程仓库，git remote add gitee git@xxxxxxxx/learngit.git

4、push 到 github 仓库， git push github master 

5、push 到 gitee 仓库，git push gitee master 

添加 ssh public key，git bash 
执行  ssh-keygen，生成 id_rsa.pub，路径在 ~/.ssh/id_rsa.pub
把该内容复制到 github 项目设置的 ssh_key，设置好
然后可以愉快执行git push啦
